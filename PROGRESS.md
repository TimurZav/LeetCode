## 📝 Ключевые ошибки и уроки

### Sliding Window
❌ **Ошибка:** Забываю отслеживать минимум внутри while
✅ **Решение:** Обновлять `min_len` внутри while после сжатия

❌ **Ошибка:** Использую `sum()` вместо переменной → O(n²)
✅ **Решение:** Держать `current_sum`, обновлять при ±элемент

❌ **Ошибка:** Путаю фиксированное и динамическое окно
✅ **Решение:** Фиксированное = цикл for, динамическое = while внутри

---

### Two Pointers
❌ **Ошибка:** Двигаю оба указателя вместо анализа
✅ **Решение:** Жадный выбор - двигать указатель с меньшим значением

❌ **Ошибка:** Неправильное условие выхода из цикла
✅ **Решение:** `while left < right` для сходящихся

---

### Hash Map
❌ **Ошибка:** Не учитываю количество повторений
✅ **Решение:** Counter вместо `{i: 1 for i in arr}`

❌ **Ошибка:** Перебор вместо HashMap → O(n²)
✅ **Решение:** Словарь для O(1) lookup

## 🎓 Изученные концепции

### Алгоритмическая сложность
- ✅ Big-O нотация: O(1), O(n), O(n log n), O(n²)
- ✅ Разница Big-O vs реальная скорость (n/2 всё равно O(n))
- ✅ Space complexity: in-place O(1) vs O(n)
- ✅ Амортизированная сложность

### Python специфика
- ✅ `set()` для O(1) проверки вхождения
- ✅ `dict.get(key, default)` безопасный доступ
- ✅ `Counter` из collections
- ✅ `defaultdict` для группировки
- ✅ `float('inf')` для инициализации
- ✅ List comprehension vs генераторы
- ✅ `enumerate()` вместо range(len())

### Паттерны
- ✅ Sliding Window (фиксированное и динамическое)
- ✅ Two Pointers (сходящиеся и одного направления)
- ✅ Hash Map для O(n) вместо O(n²)
- ✅ Жадный выбор (greedy)
- ✅ Reverse trick для массивов
